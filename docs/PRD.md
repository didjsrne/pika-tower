## 1. 개요 (Overview)

**피카타워**는 PokeAPI 데이터를 활용한 게임보이 감성의 **무한 등정(endless tower) 턴제 로그라이크** 웹 게임이다. 플레이어는 피카츄 한 마리로 무한히 높아지는 타워를 오르며, 각 층의 야생 포켓몬·보스를 턴제 전투로 쓰러뜨리고, 층 사이의 상점에서 성장 자원을 소비한다. 얼마나 높은 층까지 도달했는지가 핵심 성과 지표이며, 전 세계 플레이어와 리더보드로 경쟁한다.

### 한 줄 요약
> "피카츄 한 마리로 어디까지 오를 수 있는가" — 짧고 반복 가능한 세션 기반의 캐주얼 경쟁 로그라이크.

---

## 2. 목표 & 비목표 (Goals & Non-Goals)

### 2.1 목표
- **즉시 플레이**: 설치/로그인 없이 닉네임만 입력하면 30초 내 전투 시작.
- **짧은 세션 루프**: 한 판이 수 분 내로 끝나고, "한 판 더" 욕구를 자극하는 난이도 곡선.
- **명확한 성장감**: 전투 → 보상 → 상점 구매 → 더 강해짐의 루프가 매 층 체감되어야 함.
- **경쟁 동기**: 도달 층수 기반 글로벌 리더보드로 재도전 동기를 부여.
- **레트로 정체성**: 게임보이(DMG) 외형과 상태별 BGM으로 일관된 무드 제공.
- **모바일 우선 + 설치형**: 모바일 브라우저에서 쾌적하게 동작하고 PWA로 설치 가능.

### 2.2 비목표 (Non-Goals)
- 계정 시스템 / 소셜 로그인 / 친구 기능.
- 다회차 메타 진행(영구 업그레이드, 해금 트리) — 매 판은 동일한 시작점에서 출발.
- 멀티플레이어 실시간 대전.
- 피카츄 외 플레이 가능 캐릭터 선택, 진화, 파티 편성.
- 수익화(광고/결제). 현재는 비상업 토이 프로젝트.

## 3. 타깃 사용자 (Personas)

- **추억의 포켓몬 팬**: 포켓몬/게임보이 향수를 가진 캐주얼 유저. 익숙한 타입 상성·기술명에 즐거움을 느낌.
- **점수 경쟁러**: 리더보드 상위권을 노리고 최적 루트를 반복 시도하는 유저.
- **킬링타임 유저**: 모바일에서 잠깐씩 한두 판 즐기는 라이트 유저.

---

## 4. 핵심 사용자 흐름 (Core User Flow)

1. **진입 → 닉네임 입력 게이트**: 첫 화면에서 닉네임(최대 12자)을 입력한다. 이 입력 제스처가 동시에 브라우저 오디오 자동재생 잠금을 해제한다.
2. **타이틀 화면**: 로고, 플레이어 닉네임, 최고 기록(localStorage)을 표시. `A`로 게임 시작.
3. **전투 (층)**: 야생/보스 포켓몬과 턴제 전투.
4. **승리 → 상점**: 자금/경험치 획득 → 포켓몬 센터 상점에서 아이템 구매 → "다음 층" 선택.
5. **반복**: 층이 깊어질수록 적이 강해짐(스케일링).
6. **패배 → 게임 오버**: 피카츄 HP가 0이 되면 종료. **닉네임 + 최종 층수 + 피카츄 레벨**이 리더보드에 자동 등록되고, 리더보드 모달이 자동으로 뜬다.
7. **재도전**: `A`로 즉시 재시작(닉네임은 유지).

## 5. 기능 요구사항 (Functional Requirements)

### 5.1 시작 게이트 / 닉네임
- 닉네임 입력 필드(최대 12자), 빈 값일 때 시작 버튼 비활성화.
- 닉네임 입력 전까지 게임 키 입력(방향키/A/B)은 비활성화되어, 닉네임에 포함된 글자가 게임을 오작동시키지 않아야 한다.
- 입력한 닉네임은 게임 오버 시 리더보드 등록과 타이틀 표시에 사용된다. 재도전 시에도 유지된다.

### 5.2 입력 체계
- **조작**: 방향키(상하좌우 커서 이동), `A`(선택/Enter), `B`(취소/Esc/Backspace).
- **물리 컨트롤**: 화면의 게임보이 D-Pad / A·B 버튼 클릭(터치)으로도 동일 입력.
- 메인 메뉴는 2×2 그리드 커서 이동 규칙을 따른다.

### 5.3 전투 시스템 (턴제)
- **플레이어 시작 스탯**: Lv5, HP 60, 공격 15.
- **메뉴**: `싸운다 / 가방 / 정보 / 도망`.
  - **싸운다**: 4개 기술 중 선택.
  - **가방**: 보유한 소비 아이템 사용(턴 소모).
  - **정보**: 피카츄의 레벨/HP/공격/경험치/지닌물건 표시(턴 미소모).
  - **도망**: 불가("타워에서는 도망칠 수 없다").
- **데미지 공식(플레이어)**: `floor((공격 × 기술위력 / 50 + 2) × 타입배수 × 지닌물건보정)`.
- **데미지 공식(적)**: `floor(적공격 × 40 / 55 + 2)`.
- **타입 상성**: 전기/물/비행 공격 타입에 대해 상성표 적용(예: 전기 → 물·비행 2배, 땅 0배). 효과 메시지("효과가 굉장했다!" 등) 표시.
- **상태 이상**: `볼부비부비`는 명중 시 **반드시(100%)** 적을 **마비**시킴(원작 Nuzzle 사양). 단 전기 무효(땅 타입 등) 상대에겐 효과 없음. 마비된 적은 매 턴 25% 확률로 행동 불가.
- **연출**: 공격 시 흔들림(shake) 애니메이션, HP 바 색상(녹/황/적) 변화, 보스 등장/레벨업 배너.

#### 피카츄 기술 (고정 4종)
| 기술 | 위력 | 타입 | 효과 |
|---|---|---|---|
| 10만볼트 | 90 | 전기 | — |
| 둥실둥실풀 | 85 | 비행 | — |
| 참방참방서핑 | 85 | 물 | — |
| 볼부비부비 | 20 | 전기 | 마비(100%) |

### 5.4 적 / 층 / 스케일링
- **적 데이터**: PokeAPI에서 도감번호로 이름/타입/스프라이트를 가져옴(앞·뒤 스프라이트).
- **일반 층**: 약~중간 포켓몬 풀에서 랜덤 등장.
- **보스 층**: 10·20·30…층마다 전설/거물 포켓몬 등장. 보스 풀을 한 바퀴 돌면 다시 처음부터(난이도만 상승).
- **적 스탯 스케일링** (`floor` = 현재 층):
  - 레벨 = `4 + floor (+6 보스)`
  - HP = `(45 + floor×12) × tier × (보스 1.9)`
  - 공격 = `(8 + floor×1.4) × tier × (보스 1.45)`
  - `tier`: 보스 풀을 한 바퀴(=300층) 돌 때마다 +0.6 추가 난이도.
- **로딩 상태**: 적 데이터 로딩 중에는 입력을 막고 "데이터 로딩 중..." 표시.

### 5.5 성장 (경험치 / 레벨업)
- **승리 경험치**: `(적레벨×1.8 + floor) × (보스 2.2)`.
- **레벨업 1회**: HP 최대치 +20(+현재 HP +20), 공격 +6.
- **다음 레벨 필요 경험치**: `8 + level×4`. 한 번에 여러 레벨업 가능.

### 5.6 상점 (포켓몬 센터)
- 전투 승리 후 진입. 커서 상하 이동, `A` 구매, `B`로 "다음 층" 바로가기.
- 자금 부족/중복 보유(지닌물건) 시 구매 거부 메시지.
- "다음 층 진입" 선택 시 다음 전투 시작.

#### 상점 아이템
| 아이템 | 가격 | 종류 | 효과 |
|---|---|---|---|
| 상처약 | 120 | 소비 | HP 50 회복 |
| 고급상처약 | 320 | 소비 | HP 완전 회복 |
| 이상한사탕 | 260 | 즉시 | 즉시 레벨 +1 |
| 전기구슬 | 520 | 지닌 | 전기 기술 위력 2배 |
| 부적금화 | 420 | 지닌 | 획득 자금 1.5배 |
| 구애안경 | 640 | 지닌 | 모든 기술 위력 1.3배 |
| 먹다남은음식 | 460 | 지닌 | 매 턴 최대 HP의 6% 회복 |

- **자금 보상**: `(120 + floor×25) × tier × (보스 2.6)`. 부적금화 보유 시 ×1.5.

### 5.7 게임 오버 & 리더보드
- 피카츄 HP 0 → 게임 오버 화면(닉네임, 도달 층수, 피카츄 레벨, 신기록 여부, 최고 기록).
- **점수 등록**: 게임 오버 시 `닉네임 / 최종 층수(floor) / 피카츄 레벨(level)`을 백엔드에 자동 POST.
- **리더보드 표시 위치**: (1) 게임 오버 후 자동 모달, (2) 게임기 하단 🏆 버튼으로 언제든 호출.
- **정렬**: 층수 내림차순 → 레벨 내림차순 → 먼저 달성한 순. 상위 100개.
- 방금 등록한 내 기록은 모달에서 강조 표시(나).
- **최고 기록**: 로컬(localStorage `pika-best`)에 별도 보관, 타이틀/게임오버에 표시.

### 5.8 사운드 (BGM)
- 상태별 트랙: 타이틀 / 야생 전투 / 보스 전투 / 야생 승리 / 보스 승리 / 상점.
- 게임 오버는 무음.
- 같은 곡이면 끊김 없이 유지, 다른 곡일 때만 교체. 볼륨 0.35, 루프.
- 우상단 🔊/🔇 음소거 토글. 자동재생 정책 준수(첫 사용자 제스처 이후 재생).

## 6. 비기능 요구사항 (Non-Functional Requirements)

- **반응형 / 모바일**: 게임 화면은 450px 기준 디자인을 화면 폭에 맞춰 균일 축소(ResizeObserver). 게임보이 본체 크기는 메뉴/상태와 무관하게 일정 유지.
- **PWA**: 매니페스트 + 서비스워커로 설치형 웹앱 동작. 아이콘/테마컬러 제공.
- **성능**: 적 스프라이트는 외부 CDN(PokeAPI sprites)에서 로드. 로딩 중 입력 잠금으로 상태 꼬임 방지.
- **접근성**: 버튼 `aria-label`, 키보드 전체 조작 가능.
- **국제화**: UI 텍스트 한국어. 닉네임은 유니코드(한글 포함) 지원.

## 7. 기술 아키텍처 (Technical Architecture)

- **프레임워크**: Next.js 15 (App Router) + React 19 + TypeScript.
- **스타일**: Tailwind CSS + 커스텀 게임보이 CSS(globals.css).
- **상태 관리**: 단일 `useGameEngine` 훅(`useRef`로 상태 보관 + `useReducer` 강제 리렌더). 페이즈: `start / battle / shop / gameover`.
- **외부 데이터**: PokeAPI(포켓몬 메타/스프라이트).
- **리더보드 백엔드**: Supabase (Vercel 통합으로 연결).
  - 접근은 서버 라우트 핸들러 `app/api/leaderboard` 경유. 키는 클라이언트에 노출되지 않음.
  - `@supabase/supabase-js` 미사용 — PostgREST REST API를 서버에서 `fetch`로 호출.
  - 키 우선순위: `SUPABASE_SERVICE_ROLE_KEY` → `SUPABASE_ANON_KEY` → `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
  - 미연동 시 503 반환 → 프론트는 graceful 안내("아직 연결되지 않았어요").
- **데이터 모델** (`public.leaderboard`):

  | 컬럼 | 타입 | 비고 |
  |---|---|---|
  | id | bigint (identity) | PK |
  | nickname | text | 1~12자 체크 |
  | floor | int | ≥ 1 |
  | level | int | ≥ 1 |
  | created_at | timestamptz | 기본 now() |

  - 인덱스: `(floor desc, level desc, created_at asc)`.
  - RLS: anon 키만으로 read/insert 가능하도록 정책 정의(스키마: `supabase/schema.sql`).
- **배포**: Vercel. 프로덕션 환경변수는 Supabase 통합으로 자동 주입.

## 8. 성공 지표 (Success Metrics)

- **재도전율**: 게임 오버 후 다시 시작하는 비율.
- **세션당 평균 도달 층수** 및 그 분포.
- **리더보드 등록 수**(고유 닉네임 / 일일 신규 기록).
- **세션 길이 / 세션 수**(킬링타임 적합성).
- (정성) 난이도 곡선 체감 — 특정 층 구간에서 사망이 과도하게 몰리지 않는지.

## 9. 알려진 제약 / 향후 과제 (Open Questions & Future Work)

- **부정 점수 방지 부재**: 점수는 클라이언트가 신고하는 값이라 위변조 가능. (현재는 토이 프로젝트라 미대응.) → 서버 검증/서명, 리플레이 검증 등 고려.
- **닉네임 중복/도배**: 동일 닉네임 다수 등록, 비속어 필터 없음.
- **리더보드 기간 구분 없음**: 전체 누적만 존재(일간/주간 랭킹 없음).
- **메타 진행 부재**: 영구 성장 요소가 없어 장기 리텐션 한계.
- **밸런스**: 스케일링 계수는 수동 튜닝 값. 데이터 기반 재조정 여지.
- **콘텐츠 다양성**: 피카츄 기술 4종 고정, 적 행동이 단순(공격/마비 회피)함.
